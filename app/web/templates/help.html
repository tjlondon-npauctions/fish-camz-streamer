Setup Guide

Network Setup

The Fish Camz Box must be on the same network as the NVR or camera. On most vessels, everything connects through the Starlink router:

Camera ──POE──► NVR (built-in POE switch)
                  │
                  ├── NVR main port ──► Starlink Router
                  │                          │
                  │                          ├── Fish Camz Box
                  │                          └── Other devices
                  │
                  └── Internal POE ports (cameras only)

Key point: The NVR's built-in POE switch creates a separate internal network for cameras. The Fish Camz Box cannot see the cameras directly — it must connect to the NVR's IP address to access the video stream.

Connecting to a Uniview NVR

If you have a Uniview NVR (e.g. NVR501-04B-P4-IQ), use this RTSP URL format:

rtsp://admin:PASSWORD@NVR_IP:554/unicast/c1/s0/live
PartMeaningExample
adminNVR usernameadmin
PASSWORDNVR admin passwordSet during NVR first-run
NVR_IPNVR's IP on your network192.168.1.64
c1Channel number (POE port)c1 = port 1, c2 = port 2, etc.
s0Stream types0 = main (high quality), s1 = sub (lower)

For example, a camera on NVR port 1 with NVR IP 192.168.1.64:

rtsp://admin:MyPassword123@192.168.1.64:554/unicast/c1/s0/live
Recommended: Set H.264 on the NVR

By default, Uniview cameras output H.265 (HEVC) at 5MP. This works, but forces the Fish Camz Box to re-encode the video (high CPU, lower quality). For best results:

  1. Open the NVR web admin: http://NVR_IP in a browser
  2. Go to Settings → Camera → Video (or Encoding)
  3. Change the Main Stream video codec from H.265 to H.264
  4. Optionally reduce resolution to 1920×1080 (still excellent quality)
  5. Click Save
SettingWith H.265 (default)With H.264 (recommended)

After changing this, restart the stream from the Dashboard. The logs should show can_copy=True.

NVR Local Recording

The NVR continues to record to its own hard drive regardless of the live stream. Fish Camz taps into the NVR's RTSP output — it doesn't affect the NVR's own recording or any other functions.

This means you have:

  • Local recording on the NVR (accessible via NVR app/web UI)
  • Live stream via Fish Camz to Cloudflare (viewable on your website)
Connecting Directly to a Camera (No NVR)

If you're connecting the Fish Camz Box directly to a camera (no NVR), both devices must be on the same network. Common RTSP URLs:

BrandRTSP URL
Univiewrtsp://IP:554/unicast/c1/s0/live
Hikvisionrtsp://IP:554/Streaming/Channels/101
Dahuartsp://IP:554/cam/realmonitor?channel=1&subtype=0
Reolinkrtsp://IP:554/1
Genericrtsp://IP:554/stream1

Use the Test Connection button in Settings to verify the URL works before starting the stream.

Troubleshooting
Can't find the NVR/camera IP
Use the Scan Network button in Settings, or check your router's connected devices list.
Test Connection fails with "Connection refused"
Check the IP is correct, the device is powered on, and RTSP is enabled in the NVR settings (port 554).
Test Connection fails with "Unauthorized"
The username or password is wrong. Check the NVR's admin credentials.
Stream works but CPU is very high (>50%)
The camera is outputting H.265. Change to H.264 in the NVR settings (see above).
Dashboard shows "Stalled" or "Slow"
The Fish Camz Box can't keep up with transcoding. Reduce the resolution to 720p in Settings, or switch the camera to H.264.
Stream keeps restarting
Check the Logs for error messages. Common causes: incorrect RTSP URL, network instability, Cloudflare stream key expired.
